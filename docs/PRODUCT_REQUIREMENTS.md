# CoinPriceBar 产品需求文档

## 1. 产品定位

CoinPriceBar 是一个面向 macOS 的状态栏行情应用，用于在系统菜单栏中持续展示数字资产相关价格与汇率信息。

产品核心目标：

- 以菜单栏常驻方式提供低打扰行情观察能力。
- 支持多个交易所/市场来源，并允许后续继续插件化扩展。
- 支持用户通过 UI 面板完成大部分配置，而不是手改配置文件。
- 支持 macOS ARM / Intel 双架构分发。

---

## 2. 目标用户

适用用户包括：

- 高频关注数字资产价格的个人投资者。
- 同时使用多个交易所、多个市场类型的用户。
- 需要观察现货、合约、C2C 汇率的用户。
- 偏好菜单栏轻量监控，而非打开完整网页/客户端的用户。

---

## 3. 核心功能范围

### 3.1 菜单栏行情展示

应用需要支持：

- 在菜单栏标题中展示当前选定监控项。
- 在下拉菜单中展示多个监控项。
- 支持价格、涨跌幅、状态等信息显示。
- 支持不同格式模板：短格式、长格式、自定义格式。
- 支持交易所图标 / 文本前缀 / Emoji 前缀。

### 3.2 多数据源插件化

数据源需要支持插件化扩展，当前已覆盖：

- KuCoin Spot
- Binance Spot
- Binance C2C
- KuCoin Futures
- Binance Futures

插件统一要求：

- 暴露唯一 `source_name`
- 支持 `start(symbols)` / `stop()`
- 支持 `list_symbols()`
- 通过统一回调向应用层上报价格和状态

### 3.3 市场类型支持

产品需支持多市场类型并在 UI 中可选：

- 现货（Spot）
- 合约（Futures）
- C2C 汇率

当前 C2C 以汇率对形式存在，例如：

- `USDT-CNY`
- `BTC-CNY`
- `ETH-CNY`

### 3.4 监控项管理

用户需能够通过 UI：

- 添加监控项
- 删除监控项
- 选择交易所/市场类型
- 选择或输入交易对/汇率对
- 自定义显示名称
- 开启/关闭监控
- 控制是否可见
- 设定哪个监控项显示在菜单栏标题
- 拖拽调整顺序

### 3.5 UI 配置面板

应用需提供本地浏览器打开的 UI 配置面板。

配置面板职责：

- 编辑 UI 展示配置
- 编辑监控项列表
- 调整显示顺序
- 调整显示数量
- 切换语言
- 切换模板模式
- 自定义标题/菜单模板
- 预览模板渲染结果

### 3.6 图标与品牌展示

图标策略需要满足：

- 优先使用官方 Logo
- 失败后回退到本地图标
- 再失败后回退到文本/字母图标
- 菜单内图标和标题图标应分开处理

### 3.7 性能与刷新控制

应用需支持：

- 稳定 / 平衡 / 实时 / 自定义 刷新模式
- 自定义刷新频率
- 减少无必要日志输出
- 在后台运行时尽量低资源占用

---

## 4. 非功能需求

### 4.1 平台要求

- 优先支持 macOS
- 支持 Apple Silicon (arm64)
- 支持 Intel (x86_64)

### 4.2 打包分发

- 提供 `.app` 打包产物
- 提供 `.dmg` 分发包
- 支持 GitHub Release 自动发布
- 生成校验摘要（SHA256）

### 4.3 可维护性

- 数据源插件应尽量独立，降低耦合
- UI 面板 HTML 与 Python 服务逻辑分离
- 关键功能具备自动化测试
- 变更后需具备编译检查和测试验证

### 4.4 测试要求

测试至少覆盖：

- Python 逻辑单元测试
- 配置解析测试
- UI HTML 渲染测试
- DOM 级拖拽排序测试
- 新数据源注册和默认配置测试

---

## 5. 当前已识别的主要用户需求

根据现有功能演进，用户的显性需求包括：

1. 信息源插件化。
2. 增加 Binance 行情支持。
3. 增加 Binance C2C 汇率支持。
4. 增加 KuCoin / Binance 合约市场支持。
5. 通过 UI 编辑配置，而不是只编辑 `config.json`。
6. 菜单和标题支持模板化显示。
7. 可调整显示数量、显示顺序和显示字段。
8. 支持多语言。
9. 支持交易所图标和回退策略。
10. 支持拖拽排序且排序结果可持久化。

---

## 6. 当前非目标 / 暂不纳入

当前阶段暂不强制要求：

- 账户登录或私有 API 资产信息
- 交易下单能力
- iOS / Windows / Linux 原生支持
- 云端同步配置
- 自动签名与 notarization 完整企业级发布链路

---

## 7. 后续建议需求

建议后续版本考虑：

- 合约代码的人类友好名称映射（例如 `XBTUSDTM -> BTC 永续`）
- 明确显示市场类型标签（Spot / Futures / C2C）
- 更细粒度的 C2C 买入价 / 卖出价模式
- 错误状态和数据源延迟状态的更直观展示
- 发布流程中的签名与 notarization
